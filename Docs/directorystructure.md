# ディレクトリ構成

```
selenium-project/
├── app.py                 # Flaskアプリケーションのメインファイル
├── crawler.py             # クローリング処理の実装
├── bulk_apply.py         # 一括応募機能の実装
├── requirements.txt      # 依存パッケージリスト
├── prompt.txt           # GPTフィルタリング用のプロンプト設定
├── chromedriver         # Selenium WebDriver実行ファイル
├── crawled_data/        # クローリングしたデータの保存ディレクトリ
│   ├── settings.json    # アプリケーション設定ファイル
│   ├── checked_jobs.json # チェック済み案件の状態管理
│   └── jobs_*.json      # クローリング結果（日時付き）
├── logs/                # ログファイル保存ディレクトリ
├── templates/          # HTMLテンプレート
│   ├── index.html     # メインページ
│   └── login.html     # ログインページ
└── Docs/              # プロジェクトドキュメント
    └── *.md           # 各種ドキュメントファイル

# 配置ルール

1. ソースコード
   - Pythonソースファイル（.py）はプロジェクトルートに配置
   - 設定ファイルもプロジェクトルートに配置（prompt.txt等）

2. データ管理
   - クローリングデータは全て crawled_data/ に保存
   - 日時付きのJSONファイルで履歴管理
   - フィルタリング済みファイルは "_filtered" サフィックス付き

3. ログ管理
   - 全てのログファイルは logs/ ディレクトリに保存
   - 日時付きのログファイル名で履歴管理

4. Webリソース
   - HTMLテンプレートは templates/ に配置
   - 静的ファイル（CSS/JS）は static/ に配置

# 開発ガイドライン

1. ファイル命名規則
   - Pythonファイル: スネークケース（例: bulk_apply.py）
   - テンプレート: 小文字スネークケース（例: index.html）
   - 設定ファイル: 小文字（例: prompt.txt）

2. データ管理
   - クローリングデータは必ずJSONフォーマットで保存
   - タイムスタンプ付きでバージョン管理
   - フィルタリング結果は元ファイル名に "_filtered" を付加

3. エラーハンドリング
   - 全ての外部APIコールはtry-except文で囲む
   - エラーは適切にログに記録
   - ユーザーへは分かりやすいエラーメッセージを表示

4. セキュリティ
   - 認証情報は環境変数または設定ファイルで管理
   - パスワードなど機密情報は平文で保存しない
   - CSRFトークンを使用したセキュリティ対策実装
